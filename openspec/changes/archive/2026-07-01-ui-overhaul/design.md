## Context

当前 UI 为深色主题（`#1a1a2e` 底色 + `#e94560` 红色强调），卡片式布局，900×650 窗口。所有交互集中在一个双路由页面（/ + /settings）中。后端命令无需变更，本次翻新仅涉及前端。

## Goals / Non-Goals

**Goals:**
- 配色从暗红切换到明亮灰白 + 蓝色系
- 所有 UI 元素在 ~700×500 窗口内一屏展示，无滚动条
- 设置从独立路由改为弹窗
- 空状态有明确的操作引导
- 成功状态更醒目
- Enter 快捷键开始转换，Esc 关闭弹窗

**Non-Goals:**
- 不改后端 Rust 代码
- 不改变 IPC 接口
- 不改变核心转换流程

## Decisions

### 1. 新配色方案

```
旧 → 新
───   ───
--bg: #1a1a2e      → #f5f5f5  (浅灰底)
--surface: #16213e  → #ffffff  (白色卡片)
--primary: #0f3460  → #2563eb  (蓝色)
--accent: #e94560   → #2563eb  (蓝色，统一)
--text: #eee        → #1f2937  (深灰字)
--text-dim: #999    → #6b7280  (灰字)
--border: #333      → #e5e7eb  (浅边框)
--success: #4caf50  → #16a34a  (绿)
--error: #f44336    → #dc2626  (红)
```

### 3. Header 改造

- 移除 `<nav>` 导航链接（"转换"、"设置"）
- 左侧：小图标 + "AAB to APK" 文字
- 右侧：Java ●/○ 圆点 + bundletool ●/○ 圆点（带 title tooltip）+ ⚙ 齿轮按钮
- 高度：48px → 36px

### 4. 设置弹窗

- 点击 ⚙ 触发 `showSettingsDialog = true`
- 弹窗内容：bundletool.jar 路径配置 + 保存/取消 + 重新检测环境按钮
- 与现有 EnvCheckDialog / 添加证书弹窗风格一致
- SettingsView.vue 保留文件但从路由中移除；SettingsForm.vue 直接嵌入弹窗

### 5. 输出目录行

当前输出目录独占一个 section。改为：

```
<section> 输出目录区域
  <h2> + 目录 input + 浏览按钮
  <button> 开始转换
</section>

↓ 改为 ↓

<div class="output-line">
  输出至: <input ...> <button>浏览...</button>
</div>
<button> 开始转换 </button>
```

section padding 从 16px 进一步压缩到 12px。

### 6. 快捷键

在 App.vue 根级别监听键盘事件：
- `Enter`：触发 `handleConvert`（需文件已选择且环境就绪）
- `Escape`：查找当前打开的弹窗并关闭（settingsDialog / envCheckDialog / addKeystoreDialog / addDeviceConfigDialog）

各弹窗组件 emit `close` 事件统一处理。

### 7. 成功状态

`ConvertProgress.vue`：
- `result.success` 为 true 时自动展开日志
- 在日志顶部插入成功提示（绿色背景条），显示 "✓ 转换完成！" 和 APK 文件名链接
- 不再需要单独的"打开所在目录"按钮

### 8. 窗口配置

```json
"width": 700,
"height": 500,
"minWidth": 600,
"minHeight": 420
```

## Risks / Trade-offs

- **旧用户习惯**：配色变化大 → 明亮主题一次性切换，不设暗色选项（首版保持简单）
- **小窗口下长文件路径显示**：输出目录和文件路径可能截断 → 使用 `text-overflow: ellipsis` + `title` tooltip
- **快捷键冲突**：应用中不应有原生 input 接受 Enter 的场景 → 弹窗中的 input 会捕获 Enter 触发表单提交，不会冒泡到全局快捷键
